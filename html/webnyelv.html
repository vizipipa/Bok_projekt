Webszerkesztési nyelvek és technológiák

A weboldalak több “részt” használnak együtt. Van, ami a szerkezetet adja, van, ami a kinézetet, és van, ami a működést (pl. kattintás, űrlap).

Most megnézzük a legfontosabbakat: HTML, CSS, JavaScript, statikus vs. dinamikus oldalak, és a frontend/backendet.

HTML: mire való?

A HTML a weboldal “váza”. Ezzel mondod meg, hogy mi hol legyen az oldalon: cím, szöveg, kép, menü, gomb stb.

Egyszerűen: HTML = elrendezés és tartalom (címek, bekezdések, képek, linkek).

Példák (emberi nyelven)

  • Cím (pl. nagy betűs főcím a tetején)
  • Szöveg (egy bekezdés, amit elolvasol)
  • Link (amire kattintasz és másik oldalra visz)
  • Kép (logo vagy fotó)
  • Doboz (egy “tartály”, amibe több dolgot beleraksz)

Tipp: A HTML-ben érdemes logikusan felépíteni az oldalt (fejléc, tartalom, lábléc).

CSS: miért fontos a kinézet?

A CSS felel a weboldal kinézetéért: színek, betűméret, térköz, dobozok mérete, elrendezés.

Mit csinál a CSS?

  • Színt ad a háttérnek és a szövegnek
  • Beállítja a betűtípust és méretet
  • Rendezi az elemeket (pl. egymás mellé vagy alá)
  • Szebbé és átláthatóbbá teszi az oldalt

Példa: HTML nélkül nincs “tartalom”, CSS nélkül meg “csupasz” és rendezetlen lesz az oldal.

JavaScript: hogyan lesz “kattintható” az oldal?

A JavaScript a weboldal működését adja: például gombra kattintva történjen valami, ellenőrizze az űrlapot, nyíljon ki egy menü.

Tipikus JavaScript dolgok

  • Gombnyomásra változik a tartalom
  • Űrlap ellenőrzése (pl. email legyen helyes)
  • Felugró ablakok, üzenetek
  • Menük nyitása-zárása

Fontos: A JavaScript legtöbbször a böngészőben fut (frontend), de létezhet szerveren is.

Statikus és dinamikus weboldal

Van olyan oldal, ami csak “kész” tartalmat mutat (statikus), és van olyan, ami adatokat kezel, bejelentkezést tud, és változik a tartalma (dinamikus).

Statikus oldal

  • Egyszerűbb
  • Gyors betöltés
  • Ritkábban változik a tartalom

Dinamikus oldal

  • Összetettebb
  • Adatbázist használhat
  • Bejelentkezés, rendelés, admin felület lehet benne

Példa: Egy egyszerű bemutatkozó oldal gyakran statikus, egy webshop szinte mindig dinamikus.

Frontend és backend

Frontend: amit a felhasználó lát (a böngészőben).

Backend: ami a háttérben fut (szerver, adatok, logika).

Frontend példák

  • HTML (tartalom + szerkezet)
  • CSS (kinézet)
  • JavaScript (működés)

Backend példák

  • Adatok kezelése (pl. felhasználók, termékek)
  • Bejelentkezés, jogosultságok
  • Válasz küldése a böngészőnek

Tipp: Frontend = kirakat, Backend = raktár + pénztár.

Összegzés

  • HTML: szerkezet (mi hol legyen)
  • CSS: kinézet (színek, betűk, elrendezés)
  • JavaScript: működés (kattintás, interakció)
  • Statikus: egyszerű, ritkán változik
  • Dinamikus: összetettebb, adatokat kezel
  • Frontend: amit látunk
  • Backend: háttérmunka

Ha ezt érted, már jól átlátod, miből áll egy weboldal alapja.
